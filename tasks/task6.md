# Задание №6: Журнал транзакций
## Цель
Научиться выводить список записей из базы данных с фильтрацией и пагинацией. Закрепить навыки работы с ListView и формами фильтрации.
## Исходные данные
- Модель `Transaction` уже создана в `inventory/models.py`. - В базе есть несколько транзакций (созданных тестовыми данными и в результате 
выполнения заданий 4 и 5).
## Что нужно сделать
Создать страницу для просмотра всех транзакций (журнал операций). **Страница:** `/transactions/` **Список транзакций должен содержать 
колонки:** - Дата - Тип операции (человекочитаемое название, не `sale_rev`, а «Выручка от продажи») - Счёт дебета (in_guid) - Счёт кредита 
(out_guid) - Количество (если есть, иначе пусто) - Сумма - Комментарий **Фильтры:** - По дате (диапазон: от – до) - По типу операции 
(выпадающий список всех `OperationType.choices`) - По GUID (текстовое поле поиска по `in_guid` или `out_guid`) **Дополнительно:** - 
Пагинация (по 20 транзакций на страницу). - Сортировка по дате (по убыванию – новые сверху).
## Технические требования
- Использовать `ListView` с переопределением `get_queryset()` для фильтрации. - Для фильтрации использовать GET-параметры: `date_from`, 
`date_to`, `op_type`, `guid_search`. - Для вывода списка типов операций в фильтре использовать `OperationType.choices`. - В шаблоне 
выводить транзакции в виде таблицы. - Пагинацию сделать через `Paginator` или встроенную пагинацию ListView (`paginate_by = 20`).
## Ожидаемый результат
- По адресу `/transactions/` отображается таблица со всеми транзакциями. - Фильтры работают: можно выбрать период, тип операции, поиск по 
GUID. - После применения фильтров URL содержит соответствующие параметры. - Список разбит на страницы. - Данные отображаются корректно 
(дата в понятном формате, тип операции на русском).
## Подсказки
- Для получения значения `display_name` типа операции используйте: `get_operation_type_display()` (метод модели). - Пример фильтрации в 
`get_queryset()`:
  ```python
  def get_queryset(self):
      qs = super().get_queryset().order_by('-date')
      date_from = self.request.GET.get('date_from')
      date_to = self.request.GET.get('date_to')
      op_type = self.request.GET.get('op_type')
      guid_search = self.request.GET.get('guid_search')
      if date_from:
          qs = qs.filter(date__gte=date_from)
      if date_to:
          qs = qs.filter(date__lte=date_to)
      if op_type:
          qs = qs.filter(operation_type=op_type)
      if guid_search:
          qs = qs.filter(in_guid__icontains=guid_search) | qs.filter(out_guid__icontains=guid_search)
      return qs
```
- В шаблоне для фильтров используйте форму с методом GET, чтобы параметры попадали в URL.
- Для пагинации добавьте в контекст `page_obj` (стандартно для ListView с `paginate_by`).
