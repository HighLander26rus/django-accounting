# Задание №4: Форма «Приход товара» (покупка)
## Цель
Научиться создавать формы для операций, которые меняют остатки, и использовать готовые сервисные функции (`create_purchase`). Познакомиться 
с транзакционной логикой и обновлением данных.
## Исходные данные
- Модели `Klient`, `Tovar`, `Part`, `Transaction` уже созданы. - Сервисная функция `create_purchase` находится в `inventory/services.py`. - 
В базе есть тестовые поставщики (клиенты с флагом `is_self=False`), товары и партии.
## Что нужно сделать
Создать страницу для оформления прихода товара (покупка у поставщика). **Страница:** `/purchase/add/` **Форма должна содержать следующие 
поля:** 1. **Поставщик** (выбор из списка клиентов, у которых `is_self=False` – можно фильтровать) 2. **Товар** (выбор из справочника 
товаров) 3. **Партия** (выбор партии выбранного товара; если у товара нет партий, должна быть возможность создать новую партию прямо из 
формы — см. подсказки) 4. **Количество** (Decimal, положительное) 5. **Цена за единицу** (Decimal, положительная) 6. **Дата прихода** (по 
умолчанию текущая дата, но можно изменить)
## Логика работы
- При отправке формы вызывается `create_purchase` с параметрами:
  - `supplier_id` – ID выбранного поставщика
  - `tovar_id` – ID выбранного товара
  - `part_id` – ID выбранной партии (или созданной новой)
  - `quantity` – количество
  - `purchase_price_per_unit` – цена за единицу
  - `date` – дата - Функция создаст транзакцию типа `purchase`, которая увеличит остаток партии и создаст кредиторскую задолженность перед 
поставщиком. - После успешного сохранения выполняется редирект на страницу списка транзакций (или на список приходов, если сделаете 
отдельный).
## Технические требования
- Использовать `Form` или `ModelForm` (создать свою, не на основе модели, так как поля не все из одной модели). - Динамически обновлять 
список партий при выборе товара (можно через JavaScript или просто загружать страницу заново с выбранным товаром, но лучше реализовать 
AJAX-подгрузку или использовать `django-select2` — по желанию). - Предусмотреть вариант, когда у товара ещё нет партий: предложить создать 
новую партию (например, добавить чекбокс «Создать новую партию» и поля для даты партии и начальной цены, которые совпадают с данными 
прихода). Или просто запретить приход без существующей партии и дать ссылку на создание партии. - Проверка на положительность количества и 
цены. - Отображать сообщения об ошибках (если период закрыт, если партия не выбрана и т.д.). - После редиректа должно быть понятно, что 
операция выполнена (использовать `messages.success`).
## Ожидаемый результат
- Пользователь может выбрать поставщика, товар, партию, ввести количество, цену и дату. - После отправки формы в таблице `Transaction` 
появляется новая запись с типом `purchase`. - Остатки выбранной партии увеличиваются на указанное количество (можно проверить через админку 
или через отчёт остатков). - Если попытаться ввести операцию в закрытый месяц (период, для которого есть запись в `ClosedPeriod`), должна 
выдаваться ошибка.
## Подсказки
- Для получения списка партий по выбранному товару можно написать простой view, возвращающий JSON, и подгружать через AJAX. Однако для 
упрощения можно сначала выбирать товар, затем на той же странице перезагружать форму с GET-параметром `tovar_id` и передавать его в 
контекст для фильтрации партий. - Примерная структура формы:
  ```python
  from django import forms
  from inventory.models import Klient, Tovar, Part
  class PurchaseForm(forms.Form):
      supplier = forms.ModelChoiceField(queryset=Klient.objects.filter(is_self=False))
      tovar = forms.ModelChoiceField(queryset=Tovar.objects.all())
      part = forms.ModelChoiceField(queryset=Part.objects.none()) # пустой queryset, заполним динамически
      quantity = forms.DecimalField(min_value=0.001)
      price = forms.DecimalField(min_value=0.01)
      date = forms.DateField()
      def __init__(self, *args, **kwargs):
          tovar_id = kwargs.pop('tovar_id', None)
          super().__init__(*args, **kwargs)
          if tovar_id:
              self.fields['part'].queryset = Part.objects.filter(tovar_id=tovar_id)
```
- Во view нужно передавать `tovar_id` в форму при GET-запросе с параметром.
- Для обработки создания новой партии можно добавить в форму дополнительные поля `new_part_date` и `new_part_price`, и если они заполнены – создать партию перед вызовом `create_purchase`.
- `create_purchase` сама проверяет, закрыт ли период, и выбрасывает исключение. Ваше представление должно ловить его и выводить сообщение.
