# Задание №5: Форма «Продажа товара»
## Цель
Научиться создавать форму для операции, которая уменьшает остатки товара, формирует выручку и долг клиента. Использовать сервисную функцию 
`create_sale`.
## Исходные данные
- Модели `Klient`, `Tovar`, `Part`, `Transaction` уже созданы. - Сервисная функция `create_sale` находится в `inventory/services.py` 
(создаёт две транзакции: списание себестоимости и выручку с долгом). - В базе есть тестовые покупатели (клиенты), товары, партии с 
ненулевыми остатками.
## Что нужно сделать
Создать страницу для оформления продажи товара покупателю. **Страница:** `/sale/add/` **Форма должна содержать следующие поля:** 1. 
**Покупатель** (выбор из списка клиентов, у которых `is_self=False` – те, кто не является «владельцем») 2. **Товар** (выбор из справочника 
товаров) 3. **Партия** (выбор партии выбранного товара **только с положительным остатком**) 4. **Количество** (Decimal, положительное, не 
превышающее остаток выбранной партии) 5. **Продажная цена за единицу** (Decimal, положительная) 6. **Дата продажи** (по умолчанию текущая 
дата)
## Логика работы
- При отправке формы вызывается `create_sale` с параметрами:
  - `client_id` – ID выбранного покупателя
  - `tovar_id` – ID выбранного товара
  - `part_id` – ID выбранной партии
  - `quantity` – количество
  - `sale_price_per_unit` – продажная цена за единицу
  - `date` – дата - Функция создаст две транзакции:
  - `sale_cost` (себестоимость) – списание себестоимости со склада в расходы
  - `sale_rev` (выручка) – увеличение долга покупателя и признание выручки - После успешного сохранения выполняется редирект на страницу 
списка транзакций.
## Технические требования
- Использовать `Form` или `ModelForm` (создать свою, аналогично заданию 4). - Список партий должен динамически обновляться при выборе 
товара (AJAX или перезагрузка страницы с GET-параметром). Кроме того, необходимо **фильтровать партии только с положительным остатком** 
(остаток можно получить через `get_balance`). - При выборе партии можно показывать пользователю её остаток и закупочную цену (для 
информации, но не для редактирования). - Валидация: количество не может быть больше остатка выбранной партии. - Проверка на положительность 
количества и цены. - Отображать сообщения об ошибках (если период закрыт, недостаточно остатка и т.д.). - После редиректа выводить 
`messages.success` об успешной продаже.
## Ожидаемый результат
- Пользователь может выбрать покупателя, товар, партию (только с остатком), ввести количество (не больше остатка), продажную цену и дату. - 
После отправки формы в таблице `Transaction` появляются две новые записи: `sale_cost` и `sale_rev`. - Остатки выбранной партии уменьшаются 
на указанное количество (можно проверить через отчёт остатков). - Долг покупателя увеличивается на сумму продажи. - Если попытаться продать 
больше, чем есть на партии – выводится ошибка. - Если попытаться ввести операцию в закрытый месяц – ошибка.
## Подсказки
- Для получения партий с остатками можно использовать функцию `get_available_parts` из `services.py` (если вы её создали). Если нет – 
напишите её:
  ```python
  def get_available_parts(tovar_id):
      parts = Part.objects.filter(tovar_id=tovar_id)
      available = []
      for part in parts:
          balance = get_balance(f"tovar_{part.tovar_id}_{part.id}")
          if balance['quantity'] > 0:
              available.append({'id': part.id, 'name': str(part), 'balance': balance['quantity'], 'cost': part.purchase_price})
      return available 
``` 
- В форме вы можете передавать в `choices` для поля `part` только доступные партии. Но динамическое обновление 
при выборе товара всё равно нужно.
- Для валидации количества переопределите метод `clean()` формы:
   ```python def clean(self):
    cleaned_data = super().clean()
    part = cleaned_data.get('part')
    quantity = cleaned_data.get('quantity')

        balance = get_balance(f"tovar_{part.tovar_id}_{part.id}")

```
