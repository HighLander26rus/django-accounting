# Задание №10: Простые права доступа
## Цель
Научиться ограничивать доступ к определённым страницам (формам, отчётам) в зависимости от прав пользователя. Познакомиться с 
`@login_required`, `@user_passes_test` и настройкой простых ролей.
## Исходные данные
- В Django уже есть встроенная система аутентификации и группа пользователей. - Созданы формы прихода, продажи, журнал транзакций и отчёты. 
- Есть суперпользователь (администратор).
## Что нужно сделать
Реализовать простое разграничение доступа: 1. **Все пользователи должны быть аутентифицированы** для доступа к любым страницам (кроме 
страницы входа). Если пользователь не залогинен – редирект на `/admin/login/` или на вашу страницу входа. 2. **Две роли:**
   - **Менеджер** – имеет доступ ко всем формам (приход, продажа) и к отчётам.
   - **Наблюдатель** – имеет доступ только к отчётам (остатки, P&L, долги клиентов, журнал транзакций), но не может 
создавать/редактировать/удалять операции. 3. **Создание групп** через админку Django (или через миграции). Добавьте группы `Manager` и 
`Observer`. Назначьте права:
   - `Manager`: может добавлять, изменять, удалять `Transaction`, `Klient`, `Tovar`, `Part` (через формы и через админку – но можно и через 
админку, главное, чтобы формы были доступны).
   - `Observer`: может только просматривать (`view`). 4. **В представлениях** (views) проверяйте права:
   - Для `CreateView`, `UpdateView`, `DeleteView` – только для менеджеров.
   - Для `ListView` и отчётов – для всех аутентифицированных пользователей (и менеджеров, и наблюдателей). 5. **В шаблонах** скройте кнопки 
«Добавить», «Редактировать», «Удалить», если у пользователя нет прав.
## Технические требования
- Использовать декораторы `@login_required` и `@user_passes_test` (или миксины для class-based views). - Для проверки прав можно 
использовать встроенные разрешения Django (`user.has_perm()`), но для простоты достаточно проверки принадлежности к группе:
  ```python
  def is_manager(user):
      return user.is_authenticated and user.groups.filter(name='Manager').exists()
  def is_observer(user):
      return user.is_authenticated and user.groups.filter(name='Observer').exists()
```
- Для CreateView/UpdateView/DeleteView добавьте миксин:
  ```python
    from django.contrib.auth.mixins import UserPassesTestMixin

    class ManagerRequiredMixin(UserPassesTestMixin):
        def test_func(self):
            return self.request.user.groups.filter(name='Manager').exists()
```
- В шаблонах используйте `{% if user.groups.all.0.name == 'Manager' %}` для отображения кнопок.

## Ожидаемый результат

- Неаутентифицированный пользователь видит только страницу входа.
- Аутентифицированный наблюдатель видит отчёты, журнал транзакций, но не видит форм прихода/продажи и не может редактировать/удалять.
- Менеджер видит всё и может выполнять операции.
- Администратор (суперпользователь) имеет все права, даже если не добавлен в группы (можно проверить `user.is_superuser)`.

## Подсказки

- В админке создайте группы `Manager` и `Observer` (`/admin/auth/group/add/`). Назначьте права:
- Manager: `inventory | transaction | Can add/change/delete transaction`, `inventory | klient | Can add/change/delete klient` и т.д. (но для простоты можно дать права на все модели в приложении inventory).
- Observer: `inventory | transaction | Can view transaction`, аналогично для других моделей (нужно добавить соответствующие разрешения в модели, если их нет – создайте миграции с `Meta` permissions).
- Если не хотите заморачиваться с permissions, используйте проверку по группам, как описано выше.
- Для автоматического создания групп при миграции можно добавить код в `apps.py` или использовать `post_migrate` сигнал.
