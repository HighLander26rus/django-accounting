# Задание №7: Отчёт «Остатки товаров на складе»
## Цель
Научиться строить отчёты на основе агрегированных данных, использовать сервисную функцию `get_balance` для вычисления текущих остатков, 
реализовать фильтрацию и группировку.
## Исходные данные
- Модели `Tovar`, `Part`, `Transaction` уже созданы. - Функция `get_balance` из `inventory/services.py` позволяет получить текущий остаток 
партии (количество и сумму). - В базе есть партии товаров с разными остатками.
## Что нужно сделать
Создать страницу для просмотра остатков товаров по партиям. **Страница:** `/reports/stock-balances/` **Отчёт должен содержать:** - Таблицу 
со следующими колонками:
  - Товар
  - Партия (дата партии, закупочная цена – опционально)
  - Остаток (количество в штуках)
  - Сумма остатка (количество × закупочная цена) - **Фильтр по товару** (выпадающий список всех товаров, можно выбрать один или несколько, 
или все). - **Итоговая строка** (общее количество единиц товара на складе и общая сумма).
## Технические требования
- Использовать `ListView` или обычную функцию во `views.py`. - Для получения партий и расчёта остатков – пройтись по всем партиям (или 
отфильтрованным по товару) и для каждой вызвать `get_balance`. - Если остаток партии равен нулю – можно её не показывать (или показывать с 
нулём – на ваше усмотрение). - Фильтр по товару должен передаваться через GET-параметр `tovar_id` (или `tovars` для множественного выбора). 
Для множественного выбора можно использовать `MultipleChoiceField` в форме. - В таблице должна быть сортировка по товару и по дате партии 
(по желанию).
## Ожидаемый результат
- По адресу `/reports/stock-balances/` отображается таблица остатков. - При выборе товара в фильтре остатки показываются только для 
выбранного товара (или товаров). - Итоговая строка суммирует количество и сумму всех отображаемых позиций. - Данные корректно отражают 
реальные остатки, рассчитанные на основе транзакций.
## Подсказки
- В представлении можно получить все партии, отфильтровать по товару (если указан), затем для каждой партии вызвать `get_balance`. Собирать 
данные в список словарей и передавать в шаблон. - Пример:
  ```python
  from inventory.services import get_balance
  def stock_balances(request):
      tovar_id = request.GET.get('tovar_id')
      parts = Part.objects.all()
      if tovar_id:
          parts = parts.filter(tovar_id=tovar_id)
      balances = []
      total_qty = 0
      total_sum = 0
      for part in parts:
          balance = get_balance(f"tovar_{part.tovar_id}_{part.id}")
          if balance['quantity'] != 0: # показываем только ненулевые остатки
              qty = balance['quantity']
              amount = balance['amount']
              balances.append({
                  'tovar_name': part.tovar.name,
                  'part_date': part.date,
                  'part_price': part.purchase_price,
                  'quantity': qty,
                  'amount': amount
              })
              total_qty += qty
              total_sum += amount
      context = {
          'balances': balances,
          'total_qty': total_qty,
          'total_sum': total_sum,
          'tovars': Tovar.objects.all(),
          'selected_tovar': tovar_id,
      }
      return render(request, 'inventory/stock_balances.html', context)
```
- В шаблоне используйте цикл для вывода строк и выведите итоги.
- Для фильтра создайте простую форму с выпадающим списком товаров и кнопкой «Применить».
