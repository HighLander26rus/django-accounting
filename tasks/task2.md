# Задание №2: CRUD для справочника «Товары»
## Цель
Закрепить навыки создания CRUD-интерфейсов, научиться работать с ForeignKey (связь с группой товаров) и реализовать фильтрацию.
## Исходные данные
- Модели `Tovar` и `TovarGroup` уже созданы в `inventory/models.py`. - Админка показывает товары и группы. - В базе есть тестовые группы и 
товары.
## Что нужно сделать
Создать страницы для управления товарами и их группами.
### Часть 1: Группы товаров (CRUD)
- Список групп (`/tovar-groups/`) – таблица с названием группы и кнопками редактирования/удаления. - Добавление группы 
(`/tovar-groups/add/`). - Редактирование группы (`/tovar-groups/<id>/edit/`). - Удаление группы (`/tovar-groups/<id>/delete/`).
### Часть 2: Товары (CRUD)
- Список товаров (`/tovars/`) – таблица с колонками:
- Название товара
- Группа (название группы)
- Кнопки: Редактировать, Удалить
- **Фильтр по группе** (выпадающий список с группами, после выбора показываются только товары выбранной группы). - Добавление товара 
(`/tovars/add/`) – форма с полями:
- Название (обязательное)
- Группа (выбор из существующих групп, выпадающий список) - Редактирование товара (`/tovars/<id>/edit/`). - Удаление товара 
(`/tovars/<id>/delete/`).
## Технические требования
- Использовать `ListView`, `CreateView`, `UpdateView`, `DeleteView`. - Для фильтрации по группе в списке товаров использовать GET-параметр 
`?group_id=...` и переопределить `get_queryset()`. - Формы – `ModelForm`. - После операций – редирект на соответствующий список. - 
URL-маршруты для групп и товаров должны быть разными (можно разместить в `inventory/urls.py`). - Шаблоны могут быть простыми, но лучше 
использовать наследование от базового.
## Ожидаемый результат
- Доступны все операции для групп товаров. - Доступны все операции для товаров. - Фильтр по группе в списке товаров работает корректно 
(если выбрана группа – показываются только товары этой группы, иначе – все). - При удалении группы товаров, товары, принадлежащие этой 
группе, **не должны удаляться** (в модели `Tovar` поле `group` имеет `on_delete=models.PROTECT` – это запрещает удаление группы, если есть 
связанные товары). Вы должны либо изменить это поведение на `CASCADE` (создав миграцию), либо обработать ошибку и вывести пользователю 
сообщение, что группа не может быть удалена, пока в ней есть товары. Рекомендуется второй вариант – он безопаснее и правильнее.
## Подсказки
- Для фильтрации по группе:
  ```python
  class TovarListView(ListView):
      model = Tovar
      template_name = 'inventory/tovar_list.html'
      context_object_name = 'tovars'
      def get_queryset(self):
          qs = super().get_queryset()
          group_id = self.request.GET.get('group_id')
          if group_id:
              qs = qs.filter(group_id=group_id)
          return qs
      def get_context_data(self, **kwargs):
          context = super().get_context_data(**kwargs)
          context['groups'] = TovarGroup.objects.all()
          context['selected_group'] = self.request.GET.get('group_id', '')
          return context
  ```
В шаблоне списка товаров добавьте форму с выпадающим списком групп и кнопкой «Фильтровать».

Для обработки защиты от удаления группы с товарами можно в DeleteView переопределить post() и обработать исключение ProtectedError, либо использовать сообщения через messages.error() и перенаправить обратно.
