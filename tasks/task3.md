# Задание №3: CRUD для справочника «Партии товара»
## Цель
Научиться работать со связанными моделями (ForeignKey) и вычисляемыми полями (остаток партии). Закрепить навыки работы с class-based views.
## Исходные данные
- Модель `Part` уже создана в `inventory/models.py` и связана с `Tovar`. - В базе есть тестовые партии (например, «Товар А - 2020-01-01»). 
- Функция `get_balance` из `inventory/services.py` позволяет получить текущий остаток партии (количество и сумму).
## Что нужно сделать
Создать страницы для управления партиями товара: 1. **Список партий** (`/parts/`) – таблица с колонками:
   - Товар (название товара)
   - Дата партии
   - Закупочная цена (за единицу)
   - Начальное количество
   - **Текущий остаток (количество)** – вычисляется через `get_balance`
   - Кнопки: Редактировать, Удалить
   - **Фильтр по товару** (выпадающий список товаров). 2. **Добавление партии** (`/parts/add/`) – форма с полями:
   - Товар (выбор из существующих)
   - Дата (поле типа `date`)
   - Закупочная цена (Decimal)
   - Начальное количество (Decimal, не меньше 0) 3. **Редактирование партии** (`/parts/<id>/edit/`) – форма с теми же полями, 
предзаполненными. 4. **Удаление партии** (`/parts/<id>/delete/`) – страница подтверждения.
## Технические требования
- Использовать `ListView`, `CreateView`, `UpdateView`, `DeleteView`. - Фильтр по товару: аналогично заданию 2, через GET-параметр 
`?tovar_id=...`. - Для отображения текущего остатка в списке партий:
  - Импортировать `get_balance` из `inventory.services`
  - В методе `get_queryset` нельзя добавить остаток, поэтому его нужно добавить в `get_context_data` или вычислять в шаблоне, вызывая 
функцию для каждой партии. Рекомендуется в `get_context_data` создать список словарей с полями партии и остатком. - При 
добавлении/редактировании партии начальное количество и цена могут быть любыми положительными числами. **Важно:** изменение партии не 
должно автоматически влиять на уже существующие транзакции – это просто справочная информация для будущих операций. - Удаление партии 
возможно только если её текущий остаток равен нулю (то есть все единицы списаны). Если остаток не ноль, нужно вывести сообщение об ошибке и 
запретить удаление.
## Ожидаемый результат
- Страница `/parts/` показывает все партии с остатками. - Фильтр по товару работает. - Можно добавлять, редактировать, удалять партии (с 
учётом проверки остатка при удалении). - После успешного добавления/редактирования/удаления – редирект на список партий.
## Подсказки
- Для получения остатка в списке партий можно использовать:
  ```python
  from inventory.services import get_balance
  class PartListView(ListView):
      model = Part
      template_name = 'inventory/part_list.html'
      context_object_name = 'parts'
      def get_context_data(self, **kwargs):
          context = super().get_context_data(**kwargs)
          parts_with_balance = []
          for part in context['parts']:
              balance = get_balance(f"tovar_{part.tovar_id}_{part.id}")
              parts_with_balance.append({
                  'part': part,
                  'balance_qty': balance['quantity']
              })
          context['parts_with_balance'] = parts_with_balance
          context['tovars'] = Tovar.objects.all()
          context['selected_tovar'] = self.request.GET.get('tovar_id', '')
          return context
      def get_queryset(self):
          qs = super().get_queryset()
          tovar_id = self.request.GET.get('tovar_id')
          if tovar_id:
              qs = qs.filter(tovar_id=tovar_id)
          return qs
  ```
    Не забудьте добавить URL-маршруты и шаблоны.
